# Memory

Claude Code の memory は、セッション間で情報を永続化する仕組みです。

## 2種類のメモリ

|        |            **Auto Memory**             |     **CLAUDE.md**      |
| :----: | :------------------------------------: | :--------------------: |
| 作成者 |           Claude が自動記録            |   ユーザーが手動管理   |
| 保存先 | `~/.claude/projects/<project>/memory/` |  プロジェクトルート等  |
|  共有  |                個人のみ                | チーム（Git で共有可） |
|  用途  |    学習情報、デバッグ知見、パターン    |   ルール、指示、設定   |

## Auto Memory の仕組み

- MEMORY.md の先頭 200 行が毎セッション開始時に自動ロードされる
- 詳細は別ファイル（aws-credentials.md 等）に分けて MEMORY.md からリンク
- Claude はセッション中に必要に応じて別ファイルを読み込む

## CLAUDE.md との使い分け

- CLAUDE.md:「こうしてほしい」というルール（日本語で会話、uv を使う等）→ チーム全体で共有
- Auto Memory:「こういう知見がある」という学習結果（Bedrock 認証情報の問題等）→ 個人の知識

## Insight

- 200行制限がポイント
  - MEMORY.md は索引として簡潔に保ち、詳細はトピック別ファイルに分離するのが設計意図です。
  - 全部を MEMORY.md に書くと 200 行を超えた部分が読み込まれません。
- `/memory` コマンド: セッション中に `/memory` で Auto Memory の有効/無効を切り替えられます。
