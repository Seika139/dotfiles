# /loop コマンドの使い方

/loop はプロンプトやスラッシュコマンドを定期的に繰り返し実行するためのコマンドです。

## 基本構文

```bash
/loop [間隔] <プロンプト>
```

## 間隔の指定方法

| 書き方 |              意味               |
| :----: | :-----------------------------: |
|   5m   |             5分ごと             |
|   2h   |            2時間ごと            |
|   1d   |             1日ごと             |
|  30s   | 30秒ごと（最小1分に丸められる） |
|  省略  |         デフォルト 10分         |

## 使用例

1. 間隔を先頭に指定

   `/loop 5m /babysit-prs` → 5分ごとに /babysit-prs を実行

2. 末尾に every で指定

   `/loop check the deploy every 20m` → 20分ごとに「check the deploy」を実行

3. 間隔を省略（デフォルト10分）

   `/loop check the deploy` → 10分ごとに「check the deploy」を実行

## 実用的なユースケース

### デプロイの状態を定期チェック

```plain
/loop 5m デプロイのステータスを確認して
```

### テストを定期実行

```plain
/loop 10m mise run test を実行して結果を報告して
```

### PR の状態を監視

```plain
/loop 15m /review-pr 123
```

## 注意事項

- スケジュールされたタスクは 3日後に自動で期限切れになります
- 手動で止めたい場合は `CronDelete` でジョブ ID を指定して削除できます
- cron の最小粒度は1分なので、秒指定は1分に切り上げられます
- `/loop` だけ（プロンプトなし）では動作しません

## 補足

- `/loop` の内部では cron 式に変換されて `CronCreate` でスケジューリングされる仕組み。例えば 5m は `*/5 * * * *` に変換される
- スラッシュコマンド（例: `/review-pr`）をそのままプロンプトとして渡せるので、既存のスキルと組み合わせた定期実行が可能
- 先ほど作った `/codex-review` と組み合わせれば、例えば `/loop 30m /codex-review` で定期的にコードレビューサイクルを回すこともできる
