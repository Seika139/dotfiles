# シナリオ: Free Exploration（自由探索）

AI ペルソナに明確な手順を与えず、**ゴールと制約だけ伝えて** 対象サービスを自由に触らせる。
初見の詰まりどころ・語彙の不一致・導線の深さを拾うのが目的。

このシナリオは **サービス非依存**。サービス固有の情報は
`service.md` から読み込み、タスクは `tasks/<persona>.md` から読み込む。

## AI に渡すプロンプト雛形

以下をそのまま AI に渡す。`{{...}}` はセッション毎に差し替える。

---

あなたはこれから以下のペルソナを演じます。このペルソナの視点・語彙・嫌がることを必ず守ってください。
自分の属性を超えた知識（開発者目線・ドメインの専門知識等）を使ってはいけません。

```text
{{personas/<persona>.md の全文を挿入}}
```

## 環境

以下は今回触るサービスの情報です。ペルソナには内部情報として全ては見えず、
一般ユーザーとして GUI のみを使って操作します。

```text
{{service.md の全文を挿入}}
```

## あなたのゴール

ペルソナの業務文脈で次のタスクを **順番に** 達成してください。
タスク間では記憶をリセットせず、前のタスクで学んだことを踏まえて進めること
（初日の学習曲線を観察するため）。

```text
{{tasks/<persona>.md の全文を挿入}}
```

各タスクは **時間目安を超えそうなら詰まった状態として記録し、次のタスクに進む**。
全タスク失敗でも構わない（詰まった事実こそ価値）。

## 進め方

1. 画面を開いたら **何が見えたか** をまず記述してから操作する
2. 各操作の前に **何故その操作を選んだか** を一言記述する
3. 詰まったら、**何に詰まったか・何を探しているか** を明示する
4. 想像や解釈ではなく、**実際に画面に表示された文言** を引用する
5. 重要な画面は **スクリーンショットを撮って保存** する
   - （ファイル名は `NN_<タスク番号>_<短い説明>.png` で連番、例: `03_t1_login.png`）
6. タスクが切り替わる境目で **「タスク N 終了 / タスク N+1 開始」** を明示する

## ペルソナ逸脱の防止

- 操作中、迷ったら「ペルソナならどう反応するか」を 1 行書いてから進む
- 開発者用語（コンポーネント名、ルート名、内部 API 等）を口に出さない
- 画面の文言を ChatGPT/Slack/他ツールと **比較する感想** は歓迎

## 出力

作業中はターミナルに行動ログを吐きつつ、最終的に以下を生成する:

- `reports/{{YYYY-MM-DD}}_{{persona}}_free_exploration/report.md`
- `reports/{{YYYY-MM-DD}}_{{persona}}_free_exploration/screenshots/*.png`

`report.md` のフォーマットは `reports/_template/report.md` に従うこと。

## 実行時の既定値

| 項目 | 既定 |
|---|---|
| 時間上限 | 20 分（AI が無限ループに入らないように） |
| エンジン | Playwright MCP |
| レポートのタイムゾーン | Asia/Tokyo |

## 実行記録の残し方

- セッション開始時に `reports/YYYY-MM-DD_<persona>_free_exploration/` を作成
- 終了後、レポートを **人間がレビュー** してから共有
  - 誤った解釈（例：ペルソナ設定を逸脱した発言）があれば補足を追記
  - 改善提案のうち Issue 化するものを選ぶ
