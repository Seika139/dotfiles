# ECS (Elastic Container Service)

## 一言でいうと

Docker コンテナを AWS 上で動かすためのサービス。「コンテナのオーケストレーター」。

## 何をするもの？

Docker イメージをもとにコンテナを起動し、指定した数だけ常に動いている状態を維持する。コンテナが異常終了したら自動で再起動し、デプロイ時は新旧バージョンを入れ替える（ローリングアップデート）。

```text
ECS クラスター
  ├── auth サービス（コンテナ x 2台）
  ├── conversation サービス（コンテナ x 2台）
  └── frontend サービス（コンテナ x 2台）
```

## 主要な概念

| 概念 | 説明 |
|---|---|
| クラスター | コンテナを動かす「場所」。複数のサービスを1つのクラスターにまとめられる |
| サービス | 「このコンテナを常に N 台動かし続ける」という定義 |
| タスク定義 | コンテナの設計図。使うイメージ、CPU/メモリ、環境変数、ポートなどを記述 |
| タスク | タスク定義から実際に起動されたコンテナのインスタンス |

## 起動タイプ

コンテナを「どこで」動かすかの選択肢。

| 起動タイプ | 説明 |
|---|---|
| **Fargate** | サーバーレス。EC2 インスタンスの管理不要。推奨 → [fargate.md](./fargate.md) |
| EC2 | 自分で EC2 インスタンスを用意してその上でコンテナを動かす |

## ALB との連携

ECS サービスは ALB のターゲットグループに登録できる。ALB がリクエストを受けて、ECS のコンテナに振り分ける。

```text
ブラウザ → ALB → ECS サービス（コンテナ）
             ↑        ↑
         パスで振り分け   ヘルスチェックで
                        正常なコンテナのみに配信
```

## デプロイの仕組み

新しいイメージをデプロイすると：

1. 新バージョンのタスクが起動
2. ALB のヘルスチェックが通ったら、トラフィックを新タスクに切り替え
3. 旧バージョンのタスクを停止

ダウンタイムなしでデプロイできる（ローリングアップデート）。

## 料金

- ECS 自体は無料
- Fargate の場合: vCPU とメモリの使用量に応じた従量課金
- EC2 の場合: EC2 インスタンスの料金
