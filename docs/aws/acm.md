# ACM (AWS Certificate Manager)

## 一言でいうと

HTTPS に必要な SSL/TLS 証明書を **無料で** 発行・管理してくれるサービス。

## 何をするもの？

ブラウザで `https://` のサイトにアクセスすると、通信が暗号化される。この暗号化には「SSL/TLS 証明書」が必要で、ACM はこの証明書を発行してくれる。

```text
ブラウザ ──(HTTPS 暗号化通信)──→ ALB
                                  ↑
                            ACM の証明書で
                            暗号化を実現
```

## なぜ必要？

1. **HTTPS の実現**: 証明書がないと HTTPS 通信ができない
2. **無料**: ACM で発行した証明書は ALB や CloudFront で使う限り無料
3. **自動更新**: 証明書の有効期限（通常13か月）が来ると自動で更新される。手動の更新作業は不要
4. **EntraID の要件**: Microsoft Entra ID は `localhost` 以外で HTTP のコールバック URL を拒否する。HTTPS が必須

## 証明書の発行方法

ACM で証明書を発行するには「このドメインは本当にあなたのものですか？」という検証が必要。2つの方法がある。

### DNS 検証（推奨）

Route 53 に特定の CNAME レコードを追加して所有権を証明する。

```text
1. ACM に「spark.cygames.jp の証明書がほしい」とリクエスト
2. ACM が「この CNAME レコードを DNS に追加してください」と返す
3. Route 53 に CNAME レコードを追加
4. ACM が確認 → 証明書が発行される
5. 以降の更新も自動（CNAME が残っている限り）
```

### メール検証

ドメインの管理者メールアドレス（`admin@cygames.jp` 等）に確認メールを送り、承認リンクをクリックする方式。Route 53 が使えない場合の代替手段。

## リージョンに注意

| 用途                  | 証明書を作るリージョン                       |
| --------------------- | -------------------------------------------- |
| ALB にアタッチ        | ALB と同じリージョン（例: `ap-northeast-1`） |
| CloudFront にアタッチ | **必ず `us-east-1`**（バージニア北部）       |

CloudFront で使う場合は東京リージョンではなく `us-east-1` で発行する必要がある。

## 料金

- **無料**（ALB / CloudFront / API Gateway で使用する場合）
- インポートした外部証明書の管理も無料
