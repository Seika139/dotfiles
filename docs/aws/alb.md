# ALB (Application Load Balancer)

## 一言でいうと

アクセスを受け付けて、適切なサーバーに振り分ける「受付係」。

## 何をするもの？

ユーザーがブラウザで `https://spark.cygames.jp` にアクセスすると、その通信はまず ALB に届く。ALB は URL のパスを見て、どのサーバー（ECS のコンテナ）に転送するかを決める。

```text
ブラウザ
  ↓
ALB（受付係）
  ├── /v1/auth/*    → 認証サーバー
  ├── /v1/chat/*    → 会話サーバー
  └── それ以外      → フロントエンド
```

## なぜ必要？

1. **振り分け**: 1つのドメインで複数のサービスを動かせる（パスベースルーティング）
2. **負荷分散**: 同じサービスのサーバーが複数台ある場合、アクセスを均等に配る
3. **ヘルスチェック**: サーバーが正常に動いているか定期的に確認し、異常なサーバーには振り分けない
4. **HTTPS 終端**: ACM 証明書をアタッチすると、ALB が HTTPS 通信を処理する。背後のサーバーは HTTP のままで OK

## 図で見る位置づけ

```text
インターネット → [ALB] → [ECS コンテナ群]
                  ↑
              ACM 証明書を
              ここにアタッチ
```

## よく使う用語

| 用語 | 意味 |
|---|---|
| リスナー | ALB が待ち受けるポートとプロトコル（例: HTTPS/443） |
| ターゲットグループ | 振り分け先のサーバー群 |
| ルール | 「このパスならこのターゲットグループへ」という条件 |
| ヘルスチェック | サーバーが生きているか確認する仕組み（例: `/health` に GET） |
