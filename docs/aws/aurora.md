# Aurora

## 一言でいうと

AWS が作った高性能なリレーショナルデータベース。PostgreSQL / MySQL と互換性がありながら、最大 5 倍の性能と高い可用性を持つ。

## RDS との違い

Aurora は RDS の一種だが、ストレージアーキテクチャが根本的に異なる。

```text
【通常の RDS】
DB インスタンス ── EBS ボリューム（1つの AZ）

【Aurora】
DB インスタンス ── Aurora ストレージ（3つの AZ に自動分散、6つのコピー）
```

| | RDS (PostgreSQL) | Aurora (PostgreSQL 互換) |
|---|---|---|
| ストレージ | EBS（単一 AZ） | 3 AZ に 6 コピー。自動修復 |
| フェイルオーバー | マルチ AZ で 60〜120秒 | **30秒以下** |
| リードレプリカ | 最大 15 台 | 最大 15 台（レプリカラグが小さい） |
| ストレージ拡張 | 手動で EBS サイズを変更 | **自動**（10GB 単位で最大 128TB） |
| バックアップ | スナップショット | 連続バックアップ（任意の時点に復元可能） |
| 料金 | 安い | RDS より 約 20〜30% 高い |

## Aurora Serverless v2

インスタンスクラスを選ぶ代わりに、負荷に応じて **自動でスケーリング** する構成。

```text
低負荷時:  0.5 ACU（ほぼ最小構成）
高負荷時: 16 ACU（自動でスケールアップ）
          ↑ 秒単位で増減
```

ACU (Aurora Capacity Unit) = 約 2GB メモリ相当の処理能力。

## いつ Aurora を選ぶ？

| シナリオ | 選択 |
|---|---|
| 開発/検証環境、小規模サービス | RDS で十分（コスト優先） |
| 本番で高可用性が必要 | Aurora 推奨 |
| 負荷の変動が大きい | Aurora Serverless v2 |
| ダウンタイムを最小限にしたい | Aurora（フェイルオーバーが速い） |
