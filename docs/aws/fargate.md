# Fargate

## 一言でいうと

ECS のコンテナを **サーバーなし** で動かす仕組み。EC2 インスタンスの管理が一切不要。

## 何をするもの？

通常、コンテナを動かすには「コンテナを載せるサーバー（EC2）」が必要。Fargate を使うと、そのサーバーの管理を AWS に任せられる。OS のパッチ当て、スケーリング、セキュリティ更新、すべて不要。

```text
【EC2 起動タイプ】
EC2 インスタンス（自分で管理）
  └── コンテナ

【Fargate】
??????（AWS が管理。見えない）
  └── コンテナ ← ここだけ気にすればいい
```

## なぜ Fargate を選ぶ？

| 観点 | EC2 起動タイプ | Fargate |
|---|---|---|
| サーバー管理 | 必要（OS パッチ、スケーリング等） | **不要** |
| 起動速度 | EC2 の起動を待つ | コンテナだけ起動（速い） |
| コスト | 常時起動なら安い | 使った分だけ（短時間タスクに有利） |
| スケーリング | EC2 の台数を意識する必要あり | タスク数を増やすだけ |

## Fargate Spot

通常の Fargate より **最大 70% 安い** 代わりに、AWS の都合で中断される可能性がある。中断されてもよいワークロード（バッチ処理等）や、stg 環境でのコスト削減に有効。

```text
cdk.json の設定例:
  stg: fargateSpotPercentage: 100（全部 Spot）
  prd: fargateSpotPercentage: 50 （半分 Spot）
```

## 料金の目安（東京リージョン）

| リソース | 単価 |
|---|---|
| vCPU | 約 $0.05/時間 |
| メモリ | 約 $0.005/GB/時間 |

例: 0.25 vCPU + 0.5 GB メモリのタスクを24時間動かすと約 $1.3/日。
