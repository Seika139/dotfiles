# Lambda

## 一言でいうと

コードを書いてアップロードするだけで、サーバーなしで関数を実行できるサービス。

## 何をするもの？

イベント（API リクエスト、S3 へのアップロード、スケジュール等）をトリガーに、書いたコードが自動実行される。サーバーの管理は一切不要。

```text
トリガー                 Lambda 関数           結果
─────────               ────────             ────
API Gateway のリクエスト → コード実行         → レスポンス返却
S3 にファイルアップロード → コード実行         → DB に書き込み
毎日 9:00（EventBridge）  → コード実行         → レポート送信
```

## ECS / Fargate との違い

| | Lambda | ECS / Fargate |
|---|---|---|
| 実行モデル | 関数単位（1リクエスト = 1実行） | コンテナが常時起動 |
| 最大実行時間 | **15分** | 制限なし |
| 起動速度 | コールドスタートあり（数百ms〜数秒） | コンテナ起動後は常時応答 |
| 向いている用途 | 短時間の処理、イベント駆動 | Web サーバー、長時間処理 |
| 料金モデル | 実行時間 x メモリ（使った分だけ） | vCPU x メモリ（起動中ずっと） |
| コンテナ管理 | 不要 | Docker イメージの管理が必要 |

## 向いている用途

- API のバックエンド（API Gateway + Lambda）
- S3 にアップロードされた画像のリサイズ
- 定期実行のバッチ処理（日次レポート等）
- CloudFormation カスタムリソース
- Secrets Manager のシークレットローテーション

## 向いていない用途

- 15分を超える処理
- WebSocket / SSE の長時間接続
- 常時高トラフィックのサービス（コスト面で ECS の方が有利）

## 主要な概念

| 概念 | 説明 |
|---|---|
| ハンドラー | Lambda が呼び出すエントリポイント関数 |
| ランタイム | 実行環境（Python 3.12, Node.js 20 等） |
| レイヤー | 共通ライブラリを関数間で共有する仕組み |
| コールドスタート | 一定時間呼ばれなかった後の初回起動が遅い現象 |
| 同時実行数 | 同時に実行できる関数の数（デフォルト上限: 1000） |

## 料金の目安

- リクエスト: 100万回あたり $0.20
- 実行時間: 1GB-秒あたり $0.0000166667
- 月100万回 x 平均200ms x 128MB メモリ → 約 $0.03/月

非常に安価。使わなければ $0。

## 関連サービス

- **API Gateway**: Lambda を HTTP API として公開
- **EventBridge**: スケジュールやイベントで Lambda をトリガー
- **S3**: ファイルアップロードをトリガーに Lambda を実行
- **Secrets Manager**: Lambda でシークレットの自動ローテーション
