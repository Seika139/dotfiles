# RDS (Relational Database Service)

## 一言でいうと

PostgreSQL や MySQL などのリレーショナルデータベースを AWS 上でマネージドに運用できるサービス。

## 何をするもの？

データベースサーバーのセットアップ、パッチ適用、バックアップ、フェイルオーバーを AWS が管理してくれる。アプリケーション側はいつも通り SQL で接続するだけ。

```text
ECS コンテナ ──(SQL)──→ RDS（PostgreSQL）
                         ↑
                    バックアップ、パッチ適用、
                    レプリケーションを AWS が管理
```

## 主要な概念

| 概念 | 説明 |
|---|---|
| DB インスタンス | データベースサーバー1台。インスタンスクラス（CPU/メモリ）を選ぶ |
| マルチ AZ | 別の AZ にスタンバイを自動作成。プライマリが落ちたら自動フェイルオーバー |
| リードレプリカ | 読み取り専用のコピー。読み取り負荷を分散できる |
| スナップショット | DB のバックアップ。手動 or 自動（日次） |
| パラメータグループ | DB の設定値をまとめたもの（`max_connections` 等） |

## インスタンスクラスの読み方

```text
db.t4g.micro
│  │   └── サイズ（micro < small < medium < large < xlarge）
│  └────── ファミリー（t: バースト, r: メモリ最適化, m: 汎用）
└───────── RDS 用プレフィックス
```

| ファミリー | 特徴 | 用途 |
|---|---|---|
| t4g | バースト可能。安い。低〜中負荷向け | 開発環境、小規模サービス |
| r6g / r7g | メモリ最適化 | 本番の高負荷 DB |
| m6g / m7g | 汎用 | バランスの取れたワークロード |
