# 日次スクリプト実行不可の調査と修正

## 概要

キャッシュがない状態でも日次スクリプトが実行されないという問題を解決します。
ロジックの透明性を高めるためのログ追加と、パス解決の堅牢化を行います。

## 変更内容

### [Component Name] bash

#### [MODIFY] [07_daily_runner.bash](file:///Users/suzukikenichi/dotfiles/bash/public/07_daily_runner.bash)

1. **パス解決の強化**: `pwd` に依存せず、より確実に絶対パスを取得するように修正します。
2. **デバッグログの追加**:
   - 実行対象のプロファイルディレクトリを表示。
   - 発見されたスクリプトファイル一覧を表示。
   - 各スクリプトの実行開始タイミングでログを出力。
3. **エラーハンドリング**: スクリプトが見つからない場合や、読み込みに失敗した場合の警告をより詳細にします。

## 検証計画

### 手動確認

1. `$BDOTDIR/daily/.cache` を削除。
2. シェルを再起動し、実行対象のスクリプト名がログに表示され、実際に実行されることを確認。
3. `DAILY_PROFILE` を存在しないものに変更し、適切な警告が出ることを確認。
