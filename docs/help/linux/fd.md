# fd コマンド

Rust で書かれた find コマンドよりも高速なファイル検索ツールです。以下のコマンドでインストールできます。

## Syntax

```bash
fd [OPTIONS] [PATTERN] [PATH]
```

find コマンドと異なり、隠しファイルと .gitignore に記載されたファイルはデフォルトで検索対象から除外されます。
これらを含めるには、`-H` または `-I` オプションを使用します。

また、fd のパターンマッチはデフォルトで正規表現を使用します。

```bash
fd -t f '^git.*\.txt$'  # git で始まり .txt で終わるファイルを検索
```

`-g` オプションを使用して、シェルのワイルドカードパターンを使用して検索することもできます。

```bash
fd -t f -g '*.txt'  # .txt で終わるファイルを検索
```

## オプション

|    オプション     | 説明                                                        |
| :---------------: | :---------------------------------------------------------- |
|      `-t f`       | ファイルのみを検索                                          |
|      `-t d`       | ディレクトリのみを検索                                      |
|      `-t l`       | シンボリックリンクのみを検索                                |
|  `-e EXTENSION`   | 指定した拡張子を持つファイルを検索（例: `-e txt`）          |
|  `-H, --hidden`   | 隠しファイルも検索対象に含める                              |
| `-I, --no-ignore` | .gitignore に記載されたファイルを検索対象に含める           |
| `-x, --exec CMD`  | 検索結果に対して指定したコマンドを実行 (例: `-x "echo {}"`) |

### exec のプレースホルダについて

プレースホルダは検索結果のファイルパスに置き換えられます

- `{}` 検索結果のファイルパス（例: `root/dir/file.txt`）
- `{.}` 拡張子を除いたファイルパス（例: `root/dir/file`）
- `{/}` ファイル名（basename）のみ（例: `file.txt`）
- `{//}` 親ディレクトリのパス（例: `root/dir`）
- `{/.}` ファイル名から拡張子を除いたもの（例: `file`）

## インストール方法

**macOS**

```bash
brew install fd
```

**Debian/Ubuntu**

```bash
sudo apt install fd-find

# Debian/Ubuntu では fd コマンドが fdfind としてインストールされるため、fd として使用するためのシンボリックリンクを作成します。
ln -s $(which fdfind) ~/.local/bin/fd

# 注意: ~/.local/bin にパスが通っていない場合は設定が必要です。
# または、より一般的な /usr/local/bin へのリンクも可能です:
# sudo ln -s $(which fdfind) /usr/local/bin/fd
```
