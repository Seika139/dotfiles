# プロデュース自走の観測ルール集 (M1-M16)

シャニマス W.I.N.G. ルートを手動プレイしながら収集した「自走化に必要な
ヒューリスティック」のカタログ。`auto_emulator.games.produce` の各層が
ここに記された前提・例外・トレードオフを参照する。

実装側で参照しているコードを **対応箇所** に明記してあるので、ルールが
変わったらまずここを更新し、対応箇所に反映する。

## 詳細

### M1 確定系ボタンは見えたら押す <a id="m1"></a>

「スキップ」「OK」「次へ」「決定」は画面遷移を進める唯一の手段。検出
できたら無条件にタップする。会話送りの中央タップ (`advance_safe`) が
当たれば消化できるケースが多い。

**例外**: 「やめる」「キャンセル」ラベルが含まれるボタンは押さない
(誤って育成終了することを避ける)。

### M2 早送り x4 を ON にする <a id="m2"></a>

会話パートの再生速度を 4 倍にするトグル。OFF 状態のときに表示される
ので、見えた時点で必ず ON にする。ON 状態のラベルは「早送り x4 ON」
なので状態が読めれば再タップしない。

**実機座標**: fractional (0.833, 0.944)

### M3 オーディション戦闘で AUTO ON + 倍速 ON <a id="m3"></a>

カード戦闘 UI (TURN N が見える画面) では `Auto OFF` と `倍速 OFF`
が右上に並ぶ。両方押せば 6 ターン分の戦闘が完全自動進行する。
**`Phase 5b` 実機検証時にこの動作で 1 オーディション完走を確認した**。

**コードからの呼び出し**: `engine.enable_battle_auto()` (auto → settle →
倍速 の順)。

### M4 特別オーディションは戦略表に従う <a id="m4"></a>

シーズン別に挑戦すべきオーディション名がある:

| シーズン | 推奨オーディション | ファン目標 |
| --- | --- | --- |
| S1 | (なし、ラジオ中心) | 1,000 |
| S2 | 夕方ワイド アイドル一番 | 10,000 |
| S3 | SPOT LIGHTのせいにして / 踊っていいとも？増刊号 | 200,000 |
| S4 | THE LEGEND / オールアイドル感謝FESTIVAL | 500,000 |

これは `SEASON_STRATEGY` 定数で表現される。`_pick_audition` は
`target_auditions` のキーワードに前方一致する候補を選び、推奨能力値
の充足率が `audition_min_stat_ratio` (default 0.6) 以上のものを採用。

### M5 体力低 + トラブル率高 → 休む <a id="m5"></a>

トラブル率 (右側ピンクバッジ) が 5% を超え、かつ体力が 50% を割って
いるときは、レッスンより休息を優先する。トラブル発生で 1 週を無駄に
するより 1 週で確実に体力回復する方が ROI が高い。

**実装**: `StrategyEngine._should_rest` が `trouble_pct >=
plan.rest_trouble_threshold` かつ `hp_pct < plan.rest_hp_threshold` を
判定。両方の閾値は `SeasonPlan` 単位で上書き可能。

### M6 「ちょっと待ってください」強制チュートリアル <a id="m6"></a>

シーズン終了直前にファンが目標未達のとき、ゲームが「ちょっと待って
ください」+「振り返りへ」+「オーディションへ」の 3 ボタンを出して
プレイヤーをオーディションに誘導する。このとき「オーディションへ」
を押す以外の選択肢がない。

**現状の対応**: `state.audition_available` が True かつ
`plan.target_auditions` が空でなければ自動で audition slot 0 を選ぶ
ことで間接的にカバー。明示的な強制チュートリアル検出は未実装。

### M7 振り返りは週コストなし、上限近接時に発動 <a id="m7"></a>

ホーム画面の「振り返り」カードは週を消費しない。スキル取得とパラ
上限突破ができるので、ステータス上限 (`SEASON_STAT_CAPS`) に近づいて
レッスン効率が落ちる前に発動する。

**実装**: `StrategyEngine._should_reflect` がシーズン上限の
`reflect_stat_proximity` (default 0.85) 以上に達したステがあれば
発動。`skill_phase=False` のシーズン (S1) では発動しない。

### M8 レッスンプレビューの最右数値はファン獲得見込み <a id="m8"></a>

レッスンカード上に表示される `+31` `+20` `+277` の 3 値は左から
**「対応ステ獲得」「2nd ステ獲得」「ファン獲得」** の順。最右の最大
値がファン量で、レッスンによっては 0 (お仕事系) もある。

**未実装**: `LessonOption.preview_fans` フィールドは型上は存在するが、
リーダー側で OCR していない。実装すると `_pick_lesson_slot` で
「ファン獲得効率が高いレッスン優先」が可能になる。

### M9 流行アイコンはオーディションのみ影響 <a id="m9"></a>

ヘッダーの「流行」アイコン (Vi / Vo / Da の 1, 2, 3 位) はオーディ
ション戦闘でのアピール値倍率に影響する。**レッスンには無関係**な
ので、流行とレッスン優先順位を結びつけてはいけない。

**現状**: 戦略テーブルが流行を考慮していないので、自走中は流行
無視で進む。将来的に audition 時のみ流行に応じた推奨レッスンを
微調整する余地あり。

### M10 シーズン 1 はラジオ収録メイン <a id="m10"></a>

S1 のファン目標 1,000 は、レッスンだけでは到達困難。`ラジオの収録`
お仕事はファン獲得効率が高く、S1 のメイン経路として推奨される。

**実装**: `SEASON_STRATEGY[1].primary_lesson_preference =
("ラジオの収録", "雑誌の撮影", "トークイベント")` の先頭。

### M11 3 択ダイアログは色だけでなく文脈で <a id="m11"></a>

旧 `sample2.yml` は「黄色 (右) を固定で選ぶ」ルールだったが、これは
シーン非依存で常に最適とは限らない。**「緊張を解く」プロンプト時に
黄色を選ぶとテンション↓を招く**ケースを観測したのが M16 の発端。

**修正後の実装**: `choose_dialog_option(prompt, options)` で
prompt キーワード (緊張 / 練習 / 休 ...) を見て選択肢を選ぶ。
マッチしないときのフォールバックは黄色 (互換)。

### M12 スワイプ > 矢印クリック <a id="m12"></a>

オーディションタブで複数カードを切り替えるとき、画面右の `›` 矢印
クリックは座標精度が出にくく不発が多い。**カード本体を右→左に
スワイプ**する方が確実。

**実装**: `audition_swipe_path()` が PNG x = 1900 → 770 のドラッグを
返し、`engine._swipe_audition_next()` が実機ドラッグを発行する。

### M13 プロデュース中はサーバー側保持 <a id="m13"></a>

プロデュース進行はクライアントローカルでなくサーバーで保存される
ため、ブラウザクラッシュやリロード後でも続きから再開できる。
**Engine 側は何もせず、再キャプチャで対応**できる。

### M14 ★差でランク条件は補正される <a id="m14"></a>

オーディションの「出場条件: アイドルランク F 以上」は推奨能力値を
下回っていても突入可能で、内部的に★差 (キャラの育成段階) が補正
として機能している可能性がある。

**現状の扱い**: `audition_min_stat_ratio=0.6` で「推奨能力値の 60%
を下回ったら見送る」というガード。実機検証で値を調整する余地あり。

### M15 テンション低下は獲得効率に影響 <a id="m15"></a>

会話パートの選択肢でテンションが上下する。低テンションはレッスン
やオーディションのパラ獲得倍率に影響する。

**現状の戦略テーブルは未参照**。将来的に `state.tension_lv` を
read してテンション低い時は休息優先、などの拡張余地あり。

### M16 黄色固定は最適でない <a id="m16"></a>

具体的な観測例: S1 で「千雪の緊張を解いてあげなきゃな」というプロデュー
サーの prompt に対し、選択肢は (にらめっこでも しよう / いつも通りの
千雪で行こう / お店での経験を 思い出そう)。文脈に最適なのは中央 (緑) の
「いつも通り」だが、黄色 (右) を選ぶとテンション↓となる。

**結論**: 黄色固定は最大公約数的な default で、3 択ダイアログでは
プロンプトキーワードに基づく文脈判定が望ましい。`DEFAULT_DIALOG_RULES`
の規則:

- `緊張 / 落ち着 / リラックス` → `いつも通り / 落ち着` を含む選択肢
- `練習 / 上達 / 頑張` → `頑張 / 本気 / 全力` を含む選択肢
- `休 / リフレッシュ / 気分転換` → `休 / ゆっくり` を含む選択肢
- マッチしないとき → 黄色 fallback (旧ルール踏襲)

## ルール追加の手順

1. 実機検証で新しいヒューリスティックを観測する
2. このファイルにルール番号 (M17, M18, ...) で追加する
3. 「対応箇所」欄に実装位置を記入する
4. 必要に応じて `SEASON_STRATEGY` / `choose_dialog_option` の規則
   テーブルを更新する
5. ユニットテストを追加してルールの挙動を固定する

ルールを否定する観測 (M16 のような) は特に貴重。**観測ベースの
ルール更新は M11 のように修正履歴を残す**。
